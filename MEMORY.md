# MEMORY.md

- 【最高优先级规则】凡是正式任务，默认执行链路必须为：用户 → main → 执行 Agent → qa → main → 用户。未经用户明确豁免，不得跳过 qa，不得由 main 直接代替执行 Agent 产出正式成果。
- 用户明确约定：专职工作必须由对应专职 Agent 负责。main 的职责是统筹、分派、跟进和汇总，不应直接代做内容创作、技术开发、运营增长或审查类工作。
- 若运行环境中对应专职 Agent 不可用，必须如实向用户说明当前无法分派；不得把 main 的直出内容表述为专职助手产出。
- 正式任务的标准协作链路已确定：用户提需求后，由 main 负责任务拆解、分派给对应执行 Agent（如 content/dev/ops），执行完成后默认交由 qa 审核，再回到 main 汇总交付给用户。
- 当前环境曾核查到 agents_list 仅开放 main，因此在无法调度 content/dev/ops/qa 时，要先说明限制，再由用户决定是否接受例外处理；但该例外不能覆盖正式任务默认流程。
